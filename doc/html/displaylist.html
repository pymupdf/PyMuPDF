DisplayList¶

DisplayList is a list containing drawing commands (text, images, etc.). The intent is two-fold: as a caching-mechanism to reduce parsing of a page, and to be used as a data structure in multi-threading where one thread parses the page and another renders pages.

A displaylist is created by new_display_list, and populated with objects from a page by running Page.run_page on a device obtained from new_list_device. Replay the list (once or many times) by calling run_display_list.

Methods¶

run_display_list (Re)-run a display list through a device.

Class API¶

class DisplayList¶
run_display_list(self, dev, ctm, cookie)¶
Parameters:
  • dev (Device) – Device obtained from new_*_device.
  • ctm (Matrix) – Transform matrix to apply to display list contents. May include for example scaling and rotation, see scale_matrix and rotate_matrix. Set to fz_identity if no transformation is desired.
  • area (BBox) – Only the part of the contents of the display list visible within this area will be considered when the list is run through the device. This does not imply for tile objects contained in the display list.
  • cookie (Cookie) – Communication mechanism between caller and library running the page. Intended for multi-threaded applications, while single-threaded applications set cookie to None. The caller may abort an ongoing page run. Cookie also communicates progress information back to the caller. The fields inside cookie are continually updated while the page is being run.
